## 🔬 烟花的化学原理

烟花是化学能向光能、热能和动能转化的艺术。每一朵绽放的烟花背后，都是一场精密的化学反应。

### 基础燃烧反应

烟花的核心是 **氧化还原反应**。氧化剂释放氧气，还原剂（燃料）与之剧烈反应，产生高温、光和气体。

**黑火药** —— 烟花的基础推进剂：

$$2\text{KNO}_3 + 3\text{C} + \text{S} \rightarrow \text{K}_2\text{S} + \text{N}_2\uparrow + 3\text{CO}_2\uparrow$$

- **硝酸钾 (KNO₃)** — 氧化剂，提供反应所需的氧
- **木炭 (C)** — 还原剂/燃料
- **硫磺 (S)** — 降低着火点，加速反应

经典配比为 **75:15:10**（硝酸钾 : 木炭 : 硫磺），这一配方已沿用千年。

### 金属盐发色原理

烟花的绚丽色彩来源于 **金属离子的焰色反应** —— 金属原子在高温下电子跃迁，回落时以特定波长的光释放能量：

| 颜色 | 金属元素 | 常用化合物 | 典型反应/发射波长 |
|:---:|:---:|:---:|:---:|
| 🔴 红色 | 锶 (Sr) | SrCO₃, Sr(NO₃)₂ | SrCl₂ → 发射 606-636 nm |
| 🟢 绿色 | 钡 (Ba) | BaCO₃, Ba(NO₃)₂ | BaCl₂ → 发射 505-535 nm |
| 🔵 蓝色 | 铜 (Cu) | CuCO₃, CuCl₂ | CuCl → 发射 420-460 nm |
| 🟡 黄色 | 钠 (Na) | NaNO₃, NaHCO₃ | Na → D线 589 nm |
| ⚪ 白/银 | 铝/镁/钛 | Al, Mg, Ti | 金属粉燃烧产生白光 |
| 🟠 橙色 | 钙 (Ca) | CaCO₃, CaCl₂ | CaCl₂ → 发射 590-620 nm |
| 💜 紫色 | 锶+铜 | SrCO₃ + CuCO₃ | 红蓝混合 |
| 🥇 金色 | 铁/碳 | Fe粉, 木炭 | 铁粉燃烧产生金色火花 |

**蓝色是最难实现的颜色** —— CuCl 分子在高温下极易分解，温度必须精确控制在 1000-1200°C 之间，过高则蓝色消失。这也是烟花行业中"蓝色之难"的由来。

### 关键化学方程式

**亮珠中的典型反应（以红色亮珠为例）：**

$$\text{KClO}_4 + 2\text{Mg} \rightarrow \text{KCl} + 2\text{MgO} + \text{光和热}$$

$$\text{SrCO}_3 \xrightarrow{\Delta} \text{SrO} + \text{CO}_2\uparrow$$

$$\text{SrO} + 2\text{HCl}_{(\text{来自}\ \text{PVC})} \rightarrow \text{SrCl}_2 \rightarrow \text{红色光}$$

> 注意：PVC（聚氯乙烯）作为氯供体，与金属氧化物反应生成金属氯化物，这些氯化物在火焰中产生鲜艳的颜色。这是现代烟花配色的核心技巧。

**铝镁合金产生银白闪光：**

$$4\text{Al} + 3\text{O}_2 \rightarrow 2\text{Al}_2\text{O}_3 + \text{大量光热}$$

$$2\text{Mg} + \text{O}_2 \rightarrow 2\text{MgO} + \text{强白光}$$

**钛粉产生银色拉尾效果：**

$$\text{Ti} + \text{O}_2 \rightarrow \text{TiO}_2 + \text{白亮火花}$$

## 🏭 烟花制造工艺

> ⚠️ **安全声明**：以下内容仅供科普教育，烟花制造属于高危行业，需持有相关资质并严格遵守国家法律法规。**严禁未经许可私自制造烟花爆竹。**

### 一、原料配制（配药）

配药是烟花制造中最危险也最关键的环节，直接决定产品的颜色、亮度、燃烧特性和安全性。

**核心原则：**

| 组分 | 作用 | 常用材料 |
|:---:|:---:|:---:|
| 氧化剂 | 提供氧气支持燃烧 | KClO₄（高氯酸钾）、KNO₃（硝酸钾）、Ba(NO₃)₂ |
| 还原剂（金属燃料） | 燃烧产生高温和光 | Al 粉、Mg 粉、Ti 粉、Fe 粉 |
| 发色剂 | 产生特定颜色 | SrCO₃（红）、BaCO₃（绿）、CuCO₃（蓝） |
| 粘合剂 | 成型固化 | 虫胶（紫胶片）、糊精、酚醛树脂 |
| 缓燃/助燃剂 | 调节燃速 | 硫磺（助燃）、碳酸钙（缓燃） |
| 氯供体 | 增强金属焰色 | PVC 粉、六氯苯、氯化石蜡 |

**配药精度**：各组分称量误差通常控制在 **±0.5%** 以内，任何偏差都可能导致燃烧异常甚至事故。

### 二、造粒（制作亮珠/星体）

亮珠（也叫"星体"或"药珠"）是烟花绽放时散射出的发光颗粒，是烟花效果的基本单元。

**工艺流程：**

1. **混药** — 将配好的药剂在防爆搅拌机中均匀混合，严禁金属工具撞击
2. **润湿** — 加入酒精或水溶性粘合剂，使药粉成为湿态
3. **滚制/切割** — 在造粒锅中滚制成球形（滚珠法），或压制成片后切割
4. **烘干** — 低温（40-60°C）缓慢烘干，避免急热引发事故

**分层亮珠** — 高端产品采用多层结构，从内到外使用不同配方，实现颜色变化效果（如：内红外绿，先看到绿色，燃到内层变红色）。

### 三、装药与压实

装药工艺是决定烟花燃烧行为的关键步骤。**压实密度直接控制燃烧速度。**

#### 核心原理：密度与燃速的关系

```
密度越高 → 药剂颗粒间空隙越小 → 火焰传播面积越小 → 燃烧越慢、越稳定
密度越低 → 颗粒间空隙越大 → 氧气渗透充分 → 燃烧越快、越猛烈
```

**这是烟花制造中最关键的工艺控制点：**

- 🟢 **压实良好**：药剂致密，燃烧平稳可控，产生预期效果
- 🔴 **压实不足（有空隙）**：火焰瞬间钻入缝隙，整体同时着火 → **爆炸**
- 🟡 **过度压实**：药剂可能压碎颗粒结构，改变燃烧特性，甚至因摩擦生热引发危险

#### 震动棒（振动压实）

在工业烟花制造中，使用 **震动压实设备（震动棒/振动台）** 来确保药剂的均匀压实：

- 将药剂装入纸筒后，通过高频振动使颗粒紧密排列
- 消除内部气泡和空隙，避免燃烧时出现不均匀
- 振动频率和时间需要精确控制，不同药剂有不同参数
- **目标**：达到理论密度的 85-95%，确保燃烧稳定可预测

> 💡 **一句话总结**：烟花的"快慢"全在一个"实"字 —— 药实则缓，药松则炸。这也是为什么烟花工厂对装药工序的安全要求最为严格。

### 四、组装

**礼花弹的组装结构（由内到外）：**

```
┌──────────────────────────────┐
│         引线 (导火索)          │
│              │                │
│    ┌────────┴────────┐       │
│    │    开爆药 (中心)   │       │
│    │  (黑火药/闪光药)  │       │
│    └────────┬────────┘       │
│         ····│····             │
│      ·· 亮 珠 层 ··          │  ← 效果药（星体）
│      ··(排列有讲究)··         │
│         ·········             │
│    ┌─────────────────┐       │
│    │   开包药 (爆散层)  │       │
│    └─────────────────┘       │
│    ┌─────────────────┐       │
│    │   外壳 (纸壳/塑壳) │      │
│    └─────────────────┘       │
└──────────────────────────────┘
```

- **开包药**：推动亮珠向四周扩散的装药，用量决定绽放直径
- **亮珠排列**：不同排列方式产生不同效果（球形排列→球形绽放，环形排列→环形效果）
- **延时引信**：控制升空后多久开爆，需要精确到 **±0.3 秒**

## 🎨 烟花产品分类与燃放效果

烟花产品按照国家标准 GB 10631 分为 A、B、C、D 四个等级，按产品形态又可分为以下主要类别：

### 一、小型产品（C/D 级，个人燃放）

| 产品 | 效果描述 | 典型规格 |
|:---:|:---|:---:|
| **吐珠** | 连续吐出彩色火球，伴有啸声 | 6-30 发 |
| **喷花** | 锥形/柱形喷射，产生扇形火花帘 | 高度 2-8m |
| **旋转** | 地面或空中高速旋转，产生彩色光环 | 地旋/天旋 |
| **线香** | 手持发光棒，产生金色火花 | 燃烧 30-120s |
| **摔炮/擦炮** | 撞击或摩擦引爆，产生脆响 | 单响 |
| **小礼花** | 升空后小范围绽放 | 直径 1-3m |

### 二、组合烟花（B/C 级，中型燃放）

组合烟花是将多个发射管固定在一个底座上，通过内部引线连接，点燃后依次或同时发射。

| 类型 | 效果 | 典型规格 |
|:---:|:---|:---:|
| **小型组合** | 25-36 发，低空绽放，适合家庭庭院 | 口径 20-25mm |
| **中型组合** | 49-100 发，中空绽放，色彩变化丰富 | 口径 25-30mm |
| **大型组合** | 100-500 发，高空绽放，可持续 30-60 秒 | 口径 30-50mm |

**效果分类：**
- **牡丹** — 球形绽放，花瓣丰满，最经典的烟花效果
- **菊花** — 细长拖尾的球形，像菊花瓣向外延伸
- **柳** — 极长的下垂拖尾，如垂柳般飘逸
- **冠（锦冠）** — 密集的小亮点，如皇冠般闪耀
- **椰子树** — 粗壮上升轨迹 + 宽大散落，形如棕榈
- **瀑布** — 大量银色/金色火花从高处倾泻，如瀑布般壮观

### 三、礼花弹（A/B 级，专业燃放）

礼花弹是烟花的最高形式，由专业持证人员使用发射筒燃放。

| 口径 | 升空高度 | 绽放直径 | 效果等级 |
|:---:|:---:|:---:|:---:|
| 2 寸 (50mm) | ~60m | ~30m | 入门级 |
| 3 寸 (75mm) | ~100m | ~60m | 常规 |
| 4 寸 (100mm) | ~130m | ~90m | 中型 |
| 6 寸 (150mm) | ~180m | ~150m | 大型 |
| 8 寸 (200mm) | ~220m | ~200m | 大型 |
| 12 寸 (300mm) | ~300m | ~300m | 特大型 |
| 16 寸 (400mm) | ~350m | ~350m | 超大型（罕见） |

**高端礼花弹效果：**
- **变色弹** — 由外向内依次变化 2-3 种颜色（如 绿→红→白）
- **多重芯弹** — 同心多层效果，如"环中环"
- **图案弹** — 笑脸、心形、五角星等特殊图案（亮珠经精确排列）
- **雷弹** — 绽放后伴随震耳爆鸣
- **锦冠菊** — 先绽放菊花，再从每个端点二次绽放

### 四、专业燃放特效

| 特效 | 描述 |
|:---|:---|
| **地面瀑布** | 悬挂在桥梁/建筑上，产生大面积银瀑效果 |
| **日景烟花** | 白天燃放，使用发烟剂产生彩色烟雾 |
| **字幕烟花** | 在钢架上排列小型烟花，点燃后显示文字 |
| **水上烟花** | 从水面发射或在水面炸开，伴有倒影 |
| **音乐编排** | 通过电子点火系统，与音乐节拍精确同步 |

**萤合落 (YingHeLuo)** 是一个专业级的烟花燃放控制与编排系统，旨在将传统烟花艺术与现代数字技术深度融合。

## 🚀 快速开始

### 环境要求

- Node.js 20+（推荐 22）
- npm 或 pnpm

### 本地开发

```bash
# 克隆项目
git clone https://github.com/H8DIDI/yingheluo-v3.git
cd yingheluo-v3

# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 局域网访问（适合平板调试）
npm run dev:lan
```

访问 `http://localhost:5173` 即可使用。

### AI 助手配置

系统集成了 AI 对话功能，用于自然语言生成燃放方案。使用前需要在界面配置 AI 服务：

1. 打开 AI 助手页面
2. 点击右上角 ⚙️ 按钮
3. 填写你的 API 地址、密钥和模型
4. 点击"保存"

支持任何兼容 OpenAI 接口的 AI 服务（OpenRouter、OpenAI、本地模型等）。

## ☁️ 部署指南

### Cloudflare Pages 部署（推荐）

1. Fork 本项目到你的 GitHub

2. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/) → Pages → 创建项目

3. 连接 GitHub 仓库，配置构建参数：
   - 构建命令：`npm run build`
   - 输出目录：`dist`

4. 创建 D1 数据库：
   ```bash
   wrangler d1 create yingheluo-v3
   ```

5. 修改 `wrangler.toml` 中的 `database_id` 为你的数据库 ID

6. 执行数据库迁移：
   ```bash
   wrangler d1 execute yingheluo-v3 --file=migrations/0001_init.sql
   wrangler d1 execute yingheluo-v3 --file=migrations/0002_create_chat_log.sql
   ```

7. 部署完成后绑定自定义域名即可

### 环境变量（可选）

如需预置 AI 配置，可在 Cloudflare Pages 中设置环境变量：

| 变量名 | 说明 | 默认值 |
|:---:|:---|:---:|
| `VITE_AI_API_URL` | AI API 地址 | `https://openrouter.ai/api/v1` |
| `VITE_AI_API_KEY` | AI API 密钥 | (空) |
| `VITE_AI_MODEL` | AI 模型名称 | `xiaomi/mimo-v2-flash:free` |

## 🤝 贡献指南

欢迎提交 Issue 和 Pull Request！

```bash
# 开发流程
git checkout -b feature/your-feature
npm run dev    # 本地开发
npm run build  # 确认构建通过
git commit     # 提交（自动安全审计）
git push origin feature/your-feature
```

> 注意：项目配置了 pre-commit 安全审计，提交时会自动检查代码中是否含有敏感信息。如审计未通过，请修改后重新提交。
